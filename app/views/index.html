Introduction to Digital Registration Services

Apply to update or create a registered title

A user-centred digital registration service that captures the right data to reduce requisitions and enable automation.

The problem

Conveyancers applying to change the register generate a lot of requisitions, especially when using the AP1 form, which is needed for many transactions.

It’s a complex, multi-purpose form wrapped in legalese guidance that’s not always understood by users.

Our users are often clerical staff at conveyancers, some of whom have learned to complete the form by rote. It’s easy to make mistakes of accuracy or omission, and most lead to requisitions.

This project took replacing the paper AP1 with a digital system as a starting point.

Aims of the service:

  • make it easier for users to provide exactly (and only) the data needed
  • reduce requisitions caused by inaccurate data or missing documents
  • save HMLR time and expense in handling paper forms, and issuing and handling requisitions
  • save customers time and expense in completing forms and managing requisitions
  • make it easier for customers to manage their applications
  • ongoing, clean the data in applications and on the register

Methods to achieve this:

  • digitised data capture
  • validated formats and data where possible
  • prompts and guidance around common requisition points
  • save and resume applications
  • enabling automation where possible
  • enabling integration of compatible applications where possible

Project so far

As an MVP, we designed an end-to-end application for Noting a lease, one of the shortest, simplest routes to changing the register. This was released as a private beta to selected early adopters.

We are now (Sept 2019) working on completing a user journey for making a transfer. This entails adding representation for each party and addresses for service.
